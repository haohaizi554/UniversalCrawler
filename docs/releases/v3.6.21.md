# Universal Crawler Pro v3.6.21

**更快启动、更稳下载、更一致的 GUI 与 WebUI。**

v3.6.21 是自 v3.6.14 以来规模最大的一次生产级质量升级。经过超过 150 次提交并合入紧急重发修复，本次版本围绕超大目录启动、下载完整性、异常恢复、双端体验、日志诊断与安全边界进行了系统性收束。它不只修复单点问题，也重新梳理了关键数据链路和运行时职责，让长时间、高频率使用更稳定、更可控。

本次修复构建已完成独立校验并正式发布，建议所有用户升级；使用超大下载目录、批量任务、HLS 视频或同时使用 GUI 与 WebUI 的用户尤其推荐。

发布状态：**已发布**（2026-07-19）。2026-07-15 的首次安装包已撤下；本页对应完成发布链路修复并重新构建的同一 v3.6.21，不另起补丁版本。

首次构建同时暴露了两个发布链路问题：Release 缺少 `latest.json` / `latest.json.sig`，且发布机累积的多代 Playwright 缓存使安装包接近 1 GiB。客户端因此按安全策略拒绝热更新。当前修复已改为只收集本项目所需的当前 Chromium、headless shell、FFmpeg 与 winldd revision，并把三项发布资产提升为默认发布命令的强制门禁。

## 本次更新亮点

### 超大目录也能快速启动

过去，当下载目录包含大量历史文件时，启动阶段的递归扫描可能长时间占用磁盘并阻塞界面。v3.6.21 移除了启动热路径上的同步全盘扫描，改用有边界的后台维护与持久化恢复账本：

- 普通任务只扫描必要层级，不再无条件递归整个下载盘。
- HLS 临时工作区只清理程序自身拥有的目录，避免扩大文件系统边界。
- 下载任务在产生文件副作用前登记精确路径，成功后立即销账；异常退出后由下一次启动定点恢复。
- 后台维护失败只记录诊断信息，不再阻塞 GUI 就绪。

对于拥有数万乃至更多文件的下载目录，启动过程将更轻、更可预期。

### 下载链路更可靠，异常后可恢复

本版本重点加固了下载线程、分片、合并和清理生命周期：

- 下载线程中的意外异常会被完整传播，不再继续合并并产出“看似成功”的损坏文件。
- 分片复用增加远端文件身份校验，降低旧分片污染新任务的风险。
- 修复 N_m3u8DL-RE、curl-cffi 与 Playwright HLS fallback 失败后的临时目录泄漏。
- 改进 Bilibili 音视频流合并、CDN 地址刷新、子进程输出排空和中止清理。
- FFmpeg 下载和 Bilibili DASH 合并会为 `.downloading` / `.merging` 临时文件显式指定 MP4 muxer，清理逻辑也识别合并临时文件。
- curl-cffi HLS 请求使用 `CurlOpt.RESOLVE` 固定已经通过公网校验的 IPv4/IPv6 DNS 结果，支持自定义端口，无法安全固定时失败关闭。
- 补齐爬虫复活、中断状态、Playwright 进程关闭和限速边界处理。

下载失败时，失败记录与 Trace ID 会保留足够上下文，方便定位、复制与重试。

### GUI 与 WebUI 体验进一步统一

下载队列、正在下载、已完成、失败列表、日志中心、配置中心和工具箱继续向同一套交互契约收敛：

- 统一页面结构、主题令牌、表格状态、卡片分区、下拉框和响应式布局。
- 完善中英文热切换，覆盖导航、设置、日志、状态栏、任务选择和更新弹窗。
- 任务选择支持 `Enter` 确认、`Esc` 取消，按钮尺寸会根据翻译文本自适应。
- 文件命名规则在任务状态转换时重新投影，既保持当前页面稳定，也让后续状态使用最新规则。
- 深浅主题、强调色、字体与缩放切换减少整页重绘和旧状态闪现。
- 日志只有在用户仍停留于尾部时才自动跟随；向上查看历史记录后不再被新日志拉回底部。
- 已完成与失败记录删除改为后台执行和乐观更新；Web 端的取消等待与文件删除整体进入 executor，不再阻塞事件循环。
- 删除与重命名按具体媒体实例和单 ID 临界区协调；删除前会冻结 mutation plan，并逐项授权最终文件、临时文件和任务目录，任一目标越界即整体取消。Web 请求被取消时，后台 worker 仍会完成状态提交与 generation 清理；旧操作被同 ID 新条目取代后自动失效，不再误停新任务、删除新文件或向 WebUI 发送错误成功事件。
- GUI 顶层全屏播放器与 Web Fullscreen backdrop 会继承并实时同步当前主题。
- 图片预览使用独立轮播状态，隐藏视频进度条/时间，统一前后切换、序号、禁用态、键盘与定时播放；失效候选不会再造成列表选中项与当前画面分裂，点击一次即可恢复。
- GUI 与 WebUI 统一识别包括 AVIF 在内的图片类型，避免一端进入轮播、另一端误交给视频播放器。
- 启动模式卡片改为具备按钮语义的可聚焦控件，支持 Tab、Enter/Space 与数字快捷键；小屏和高 DPI 环境使用可滚动内容并限制对话框不超出可用工作区。

无论使用桌面 GUI 还是浏览器 WebUI，核心操作的语义和反馈现在更加一致。

### 日志与失败记录从“能看”升级为“可诊断”

日志中心和失败列表已转为更适合长期运行的增量数据管线：

- 日志使用增量 tail、文件轮转识别、内存热缓存和持久化解析缓存。
- 解析、本地化、筛选、排序和分页在 worker 中完成，减少 UI 线程压力。
- 失败记录使用 SQLite 结构化查询，支持分页、复制、单条删除、全部清空和保留周期设置。
- 删除或清空失败记录只抑制已经在途的旧写入；同一资源之后再次失败时可以重新入库，不再形成永久黑名单或无界 ID 集合。
- 日志洪峰与进度事件支持合并、背压和有序终态交付，降低高频刷新造成的卡顿。
- GUI 与 WebUI 共用显示投影，减少语言切换后中英文混排和短暂闪烁。

### 更新与安全边界更完整

v3.6.21 完善了版本检查、更新清单和路径授权链路：

- 自动更新支持 SHA256、平台匹配和 Ed25519 签名清单。
- 未验签的 Release 只参与版本提示，不会进入自动安装流程。
- 加固更新下载的 SSRF、重定向、敏感字段过滤和会话绑定。
- Web 配置更新必须遵守受批准的保存目录边界，不能绕过路径授权。
- Playwright 公网守卫统一安装在 BrowserContext，覆盖 popup 首请求，并阻断 WebSocket、Worker、SharedWorker 与 Service Worker 绕过。
- 快手固定 profile 探针禁止 `APIRequestContext` 自动重定向；外域或内网 `Location` 不会绕过 BrowserContext 守卫。
- Bilibili 输入使用解析后的 hostname 精确匹配根域和真实子域；小红书带登录 Cookie 的入口遇到外域、伪子域或解析失败时安全回退主页。
- Requests 的显式直连映射同时关闭 HTTP、HTTPS 与 `ALL_PROXY`，不再被环境或系统代理悄然改写。
- Python DNS 故障回退缓存增加 TTL/LRU 上限；Chromium 使用带 `endpoints.ips` 的 DoH 增强引导配置，在系统 DNS 故障时仍保留 TLS/SNI 校验。
- 修正多项平台签名、参数编码与浏览器反检测边界问题。

> 自动更新依赖安装包、`latest.json`、`latest.json.sig` 三项资产同时存在；本页额外发布 Ed25519 公钥用于独立审计。当前个人发布阶段未启用 Authenticode，安装包身份由 Ed25519 签名清单中的 SHA-256 强制约束。

## 平台与功能改进

- **Bilibili**：完善 BV 号、UP 主、主页、合集、分享链接和关键词入口；改进主页跳转、音视频流获取与下载合并，并拒绝 `bilibili.com.attacker.example` 等伪子域。
- **MissAV**：提升 HLS fallback 稳定性，修复高频失败时的工作区残留；浏览器指纹使用实际 Chromium 版本，避免 UA 与 JS 引擎能力错位。
- **快手**：分享/详情链接先走 HTTP 快路，再用提前监听首屏媒体响应的静默浏览器；只有检测到明确登录入口时才打开可见登录窗口，持久化 Cookie 与 origin/localStorage 后无头重试。可见会话延迟恢复 origin 存储，避免 Playwright 内部恢复页闪窗。
- **小红书**：登录入口严格限制为受信根域或真实子域，外域跳转在创建带 Cookie 的页面前回退主页。
- **抖音**：修正签名与非 ASCII 参数处理中的边界问题，强化请求与恢复流程。
- **本地媒体**：改进元数据补救、内置播放器、播放位置与 MKV 兼容处理。
- **默认打开方式**：完善 Windows 文件关联注册与生效说明。

## 升级说明

- 可直接覆盖安装；个人配置、Cookie、日志、缓存与认证文件不会写入安装包，也不会因升级被删除。
- 用户数据默认位于 `%LOCALAPPDATA%\UniversalCrawlerPro`。
- 首次启动可能执行一次有边界的旧版本临时文件迁移，不会递归扫描整个下载盘。
- 新版会自动创建失败记录与下载恢复数据库，无需手工迁移。
- 安装包内含完整浏览器与音视频工具链，文件体积较大；首次运行也可能受到 Windows Defender 或第三方安全软件扫描影响。

## 随包组件

- `UniversalCrawlerPro.exe`：桌面 GUI。
- `CrawlerWebPortal.exe`：WebUI / FastAPI 入口。
- `UCrawlLauncher.exe`：启动中心，可选择 GUI、WebUI、CLI、测试与代码量统计等模式。
- `UCrawlCLI.exe`：带真实标准输入/输出的 CLI 与交互式引导入口。
- `updater_helper.exe`：安全更新辅助程序。
- `ffmpeg.exe`、`ffprobe.exe`：音视频合并与媒体信息探测。
- `N_m3u8DL-RE.exe`：HLS 下载。
- Playwright Chromium：需要浏览器环境的采集流程。

## 使用提示

- 平台可能存在登录态、Cookie、风控、地区或网络限制，请先在配置中心确认认证与代理状态。
- 下载失败时，优先查看失败列表诊断、日志中心 Trace ID 和对应平台日志。
- 如果 Windows 在首次启动时进行安全扫描，请等待扫描完成，避免连续重复启动多个实例。
