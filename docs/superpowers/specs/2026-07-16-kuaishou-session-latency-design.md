# 快手会话延迟与登录态稳定性设计

## 背景与证据

同一快手短链任务在结构化日志中出现 27、36、43 秒的端到端耗时。23:04 样本中，控制器在 23:03:37 启动，短链规范化完成后到 23:04:00 才出现第一条平台日志；随后浏览器首屏与媒体捕获又分别消耗约 12 秒和 7 秒。有头资料页流程还会对同一目标重复执行 `goto`，失效登录态分支可能形成 `goto -> reload -> goto -> goto`。

代码没有显式清空 Cookie，但 `page.request` 与 BrowserContext 共用 Cookie Jar。服务端校验响应可以通过 `Set-Cookie` 改变浏览器 Cookie，当前逻辑随后无条件把 `context.storage_state()` 覆盖回 `ks_auth.json`，缺少关键 Cookie 防退化检查。

## 目标

- 短链展开不再等待并下载完整详情响应两次。
- 同一个页面和目标 URL 只执行一次所有者导航；登录校验不得再无条件重载有效页面。
- 有头窗口从创建到真实导航之间只保留必要的本地操作，避免长时间暴露 `about:blank`。
- 登录状态探针不得修改浏览器 Cookie Jar。
- 自动持久化不得用缺少主站 `userId` 或关键认证 Cookie 的缩水快照覆盖已有状态。
- 单条详情捕获优先消费首屏响应和 DOM 媒体，使用有界事件等待，不再用整页刷新作为常规路径。
- 保留公网 URL 逐跳校验、Context 级 Playwright 网络守卫、登录失效后的人工登录回退和停止语义。

## 方案选择

采用“最小行为修正”，不引入浏览器池或跨任务持久 Context。

1. 短链解析改用 curl_cffi 总超时与有界内容回调：整个重定向链（含公网 DNS 校验）最多 15 秒、5 跳、2 MiB；校验后按剩余预算重算传输超时，并用有界共享闸门限制悬挂解析线程。最终详情 HTML 缓存后供直连解析复用，缓存响应 4xx/5xx 或其他 HTTP 失败仍保留分享任务分类。
2. `_ensure_login` 返回已导航页面的可用状态，后续 `_navigate_to_target_page` 在当前 URL 已等价于目标时直接复用。首页搜索同样跳过重复首页导航。
3. Profile 校验使用由 Playwright 创建的独立 `APIRequestContext`，用当前 Cookie 构造请求头但不共享 Cookie Jar；请求结束必须 `dispose()`。
4. 持久化前比较新旧快照的认证 Cookie 家族：至少保留可用于 `https://www.kuaishou.com/` 的 `userId`，并把 `web_st`/`webday7_st` 视为同一可轮换家族。只有服务端确认或首次人工登录才能替换旧家族。
5. 详情捕获用 `threading.Event` 接收 response 回调信号，并以短分片轮询等待；点击播放后只做 DOM/响应复查。常规失败不再 `reload`，避免 Cookie 轮换和页面状态被重置。

未采用浏览器预热池：它会引入跨任务 Cookie 隔离、生命周期和内存回收复杂度，且不能解决短链重复 GET、重复导航和 Cookie Jar 污染这些根因。

## 数据流

```text
输入短链
  -> curl_cffi 15 秒总超时 + 逐跳公网校验
  -> 最终 URL + 可选详情 HTML 缓存
  -> HTTP 直连解析（复用缓存）
  -> 失败时启动一次详情浏览器会话

浏览器会话
  -> Context 创建并恢复登录态
  -> 页面创建
  -> 首次且唯一的目标导航
  -> 隔离 APIRequestContext 校验服务端会话
  -> 防退化持久化
  -> 响应事件/DOM 捕获媒体
```

## 错误处理与安全

- 短链响应超过 2 MiB、15 秒总预算或 5 次重定向后停止并回退浏览器，不无限缓存响应体；DNS 校验不得突破总预算或无界创建后台线程。
- 任何重定向仍由现有 `DomainPolicyEngine` 逐跳验证；不接受外域伪装或内网目标。
- 隔离登录探针不自动跟随重定向，响应异常返回“不确定”，由 DOM 兜底，不强迫重新登录。
- 防退化拒绝写回时保留原文件并记录原因，不把敏感 Cookie 值写入日志。
- 用户停止任务时，事件等待与页面等待都必须快速退出。

## 测试策略

- 单元测试覆盖短链最终响应复用、DNS 后预算重算、悬挂校验限流、缓存 4xx/5xx 回退、内容回调中止、失败分流和重定向上限。
- 单元测试覆盖资料页、首页搜索与健康 Cookie 路径不会重复导航。
- 认证测试覆盖隔离请求上下文、不共享 Cookie Jar、响应释放和缩水快照拒绝写回。
- 单条详情测试覆盖首屏响应立即完成、点击后事件完成、失败时不执行 reload。
- 运行快手单元测试、相关网络安全测试、测试套件分类契约、Ruff 与完整受影响套件。

## 非目标

- 不改变快手下载器、GUI/WebUI 展示或全局网络安全边界。
- 不持久化浏览器进程，不跨任务共享 BrowserContext。
- 不把真实 Cookie 值或请求签名写入诊断日志。
