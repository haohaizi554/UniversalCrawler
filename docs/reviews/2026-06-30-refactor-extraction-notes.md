# 2026-06-30 重构提取记录

本文合并原有零散提取笔记，去掉乱码和过时片段，只保留对当前项目仍有价值的工程结论。

## 前端适配层

前端状态统一收口到 `FrontendStateService`。GUI 和 WebUI 不再直接理解下载管理器内部结构，而是消费统一的页面字段和动作语义。这降低了 GUI/WebUI 分叉，也让高频刷新可以在适配层做合并、节流和局部更新。

## HLS 与 MissAV 下载

MissAV / HLS 链路的经验是：很多资源站 URL 直连会 403，但浏览器可通过 Referer、Cookie、Range 和本地代理语义访问。当前实现应优先保持下载器接口清晰，将本地 HLS 代理、请求头和外部 N_m3u8DL-RE 调用隔离在下载器内部，避免把平台反爬细节泄漏到 UI。

N_m3u8DL-RE 进度输出需要解析并回传给统一下载进度接口；否则 GUI 只能显示固定 50% 和 0 B/s。外部进程下载必须保留停止检查、退出码处理和日志追踪。

## Qt 前端样式

QSS 应集中在主题模块和页面级样式中，避免散落在业务逻辑里。控件样式要优先解决焦点态、下拉框弹层、滚动条、表格选中态和深浅主题一致性，不要为单个截图临时堆样式。

## 日志与高频刷新

日志中心超过千条后容易卡顿，根因通常是全量重建和不可见页面刷新。成熟策略是：当前页面只渲染可见窗口，切换 UI 最大日志数时只调整内存 ring buffer 容量，小变大不回读补齐，大变小直接丢弃缓存中的多余项。

## 文档整理结论

重构记录应放在 `docs/reviews/`，具体事故复盘放在 `docs/postmortems/`，已合并的单点修复过程放在 `docs/fixes/archive/`。同一主题只保留一个主入口，避免 README、修复记录和复盘互相复制同一段内容。
