## 2026-07-14 启动模式弹窗接入共享窗口体系

- 将实际启动路径中的模式选择弹窗从原始 `QDialog` 迁移到共享 `ChromedDialog`，统一自绘标题栏、窗口控制器、图标和主题令牌。
- `entry.dispatcher` 改为延迟导入并委托 `entry.mode_selection_ui`，消除同一弹窗的重复实现，同时保留 CLI 和无 Qt 启动能力。
- 补充启动阶段顶层窗口的工程契约与事故记录，明确禁止“系统原生标题栏 + 硬编码内容主题”的双主题真值。

## 2026-07-11 打包态启动恢复与目录清理

- 移除 `DownloadManager` 构造期间的同步全盘递归清理，改为后台启动维护与 dispatcher 门闩。
- 新增 WAL/FULL SQLite 下载恢复账本，任务创建前记录精确目录，成功立即销账，失败目录按路径去重并在使用后销账。
- 普通遗留扫描限制为两层，并使用可跨重启续跑的持久化扫描前沿；M3U8 仍只递归自己的工作目录。
- 失败记录 SQLite 统一启用 WAL、`synchronous=FULL`、busy timeout 和短连接关闭协议。
- 修正 dispatcher 退出超时误报：`Thread.join()` 后使用 `is_alive()` 判断，不再把固定返回的 `None` 当成超时。
- 增加打包态大目录启动验收与事故复盘文档。
