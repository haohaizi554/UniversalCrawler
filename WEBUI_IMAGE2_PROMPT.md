# UniversalCrawlerProplus WebUI 重构提示词（image2）

> 这个文档用于给 ChatGPT image2（或同类视觉模型）生成 WebUI 设计草图。  
> 目标：在 `app/web/static/index.html` 与桌面端 `app/ui/pages/*` 的能力统一下，输出一组**企业级、非重复、主题一致、颗粒度不重叠**的 7 个标签页界面。

---

## 一、项目分析（先给模型的上下文）

项目是一个“桌面 + Web 双端同源前端系统”，前端状态来自 `FrontendStateService`。  
Web 端当前结构是左侧固定导航 + 主内容区切页（`#leftPanel + .page-stack`）：

- `queue`：下载队列
- `active`：进行中下载
- `completed`：已完成/可预览
- `failed`：失败列表 + 诊断
- `logs`：日志中心
- `settings`：设置中心
- `toolbox`：工具箱

桌面端对应页面同样有这 7 类信息域，核心是：
- **统一入口（首页语义）**：`queue` 页面应作为“常用入口”。
- **动作与详情分离**：左边主列表/控制区，右边详情/预览/趋势区，不要把同一信息重复到多个模块。
- **状态驱动**：按钮状态、计数、分页、主题、路径均应来自状态快照，不做“手写静态内容”。

CSS 现有视觉基线（建议沿用）：
- `--bg: #f6f8fb`（浅） / `#171a1f`（深）
- `--panel: #ffffff` / `#22262d`
- `--text: #111827` / `#e5e7eb`
- `--muted: #6b7280` / `#9ca3af`
- `--border: #e5e7eb` / `#383f48`
- `--accent: #1677ff` / `#3b82f6`
- `--accent-soft: #eaf3ff` / `#1f2d46`
- 半径：卡片/控件 7~10px，按钮/输入框 7px
- 主字号：14px 起，`--ui-scale` 和 `--base-font-size` 可切换

布局约束：
- 顶栏高度约 `66px`，内边距 `10px 18px`，控件高度约 `40px`（主题/目录/按钮/搜索输入）。
- 左侧导航宽度约 `206px`。
- 主区为 `.page-stack`，页内内容多采用 `grid(1fr, minmax(320px, var(--detail-width, 360px)))` 两列结构。
- 深色浅色要一套变量驱动，切换后不改结构，只改色彩。

## 二、7 个标签页的职责（避免重复）

### 1）`queue`（常用入口）
- 显示“保存路径 + 搜索 + 启停 + 主题”这类全局入口操作。
- 左/主区：任务队列表格 + 事件条（最近任务/系统事件）+ 分页器。
- 右侧无冗余详情，避免与 `active/completed/failed` 页面重复展示。
- 内容关注：**“我将开始什么？”** 与 **“当前队列状态如何？”**

### 2）`active`
- 左：正在下载列表（进度条、速度、剩余时间、可操作按钮）。
- 右：当前任务快照/事件时间线（趋势图、速率、元数据）。
- 内容关注：**“正在发生什么，现在怎么办？”**

### 3）`completed`
- 左：已完成列表 + 分页。
- 右：媒体预览区 + 文件信息面板。
- 内容关注：**“已经下载了什么，如何复核和快速打开？”**

### 4）`failed`
- 左：失败列表 + 失败原因列。
- 右：失败详情 + 解决建议/日志片段。
- 内容关注：**“为什么失败，下一步怎么处理？”**

### 5）`logs`
- 左上：日志分类标签 + 过滤器 + 操作按钮（清空/导出/打开日志）。
- 左下：日志时间线表格。
- 右：日志条目详情 + 堆栈（可选）。
- 内容关注：**“问题发生链路（时间顺序）”**，不做重复任务列表。

### 6）`settings`
- 左：设置分类（基础/下载/平台/播放/日志/外观）导航。
- 右：当前分类说明 + 表单控件（优先下拉/开关，不建议纯文本输入）。
- 内容关注：**“可控参数”**，不同分类只显示其参数，不显示其他页重复信息。

### 7）`toolbox`
- 左：工具卡片网格 + 最近使用。
- 右：工具说明 + 打开动作。
- 内容关注：**“附加能力入口”**，与下载主链路保持边界隔离。

## 三、image2 系统提示词（可直接复制）

```text
你是高级前端界面设计助手，需要输出 7 张工业级桌面风格 WebUI 设计图（可分别对应 queue/active/completed/failed/logs/settings/toolbox），要求统一且可直接用于前端落地。

总体风格：
1) 单页应用控制台风格，不是购物/内容消费页面。
2) 左侧固定导航栏（约 206px），右侧主工作区（白/暗色卡片布局）。
3) 全局颜色与字体统一，支持浅色与深色主题切换；用 token 渐进定义：
   - 背景：#f6f8fb（浅）/ #171a1f（深）
   - 面板：#ffffff（浅）/ #22262d（深）
   - 文本：#111827（浅）/ #e5e7eb（深）
   - 次要文本：#6b7280（浅）/ #9ca3af（深）
   - 边框：#e5e7eb（浅）/ #383f48（深）
   - 强调：#1677ff（浅）/ #3b82f6（深）
   - 成功：#22c55e，警告：#f59e0b，危险：#ef4444
4) 统一控件规格：
   - 按钮/输入框/下拉框高度 34~40px，圆角 7px
   - 表头/内容区有明显分区、网格、分页，不要无意义边框和空白
   - 图标优先于文字按钮，必要时配 1 词说明
5) 全局布局：
   - 顶栏有 Source 下拉、搜索框、数量下拉、开始、停止、切换目录、主题按钮
   - 主区左为主功能表格，右为右栏详情（当页面属于“详情型”时）
   - 下方有状态条，显示运行中/下载量/上传量/完成/失败/版本

必须满足：
- 主题统一：同样控件在 7 页样式一致，不允许同页风格突变。
- 粒度对齐：同一条信息不在队列、进行中、已完成、失败之间反复重复。
- 无重复内容：同一页内也不出现“同样字段”在多个卡片重复。
- 交互克制：只显示该页需要的关键按钮，避免堆满无关操作。
- 以“可落地”为目标，不使用抽象艺术化效果（无拟物大阴影、无花哨插图）。

请生成 queue 页面首先作为默认页，然后给出其它 6 个标签页的变体（统一骨架，替换主区内容）：
- active：左边下载中表格 + 右边事件时间线/趋势
- completed：左边已完成分页 + 右边预览播放器和文件信息
- failed：左边失败列表 + 右边失败详情与建议面板
- logs：左上 tab/过滤器，左下日志表格，右边日志详情
- settings：左侧分类导航，右侧分组设置卡片，突出默认值、下拉和开关
- toolbox：左侧工具卡片网格，右侧最近记录和工具详情
- logs 和 settings 页面必须保留足够留白与可读性，避免“同一页面像打印表格”。

画面要求：
- 16:9，分辨率 2560x1440
- 无重复 UI 元素；每个标签页都能独立说明该页价值
- 输出高对比、低噪声，适合阅读长表格
- 以中文界面文案为主
```

## 四、按标签页的变体提示词（可选）

### Queue 变体
```text
同一设计系统下，渲染 queue 页面。左侧是“下载队列表”，包含标题/平台/状态/时长/操作。右上显示保存目录与刷新入口，页尾显示最近事件、分页器、总数统计。不要出现媒体预览区和失败诊断区。
```

### Active 变体
```text
同一设计系统下，渲染 active 页面。强调进度条、速度、剩余时间与当前任务控制（暂停/重试等）。右栏主打事件时间轴和性能曲线（短时趋势），不要展示已完成列表。
```

### Completed 变体
```text
同一设计系统下，渲染 completed 页面。左侧是可分页列表，右侧是视频播放器（或媒体预览占位）+ 文件元信息列表。列表与详情字段不要重复。
```

### Failed 变体
```text
同一设计系统下，渲染 failed 页面。突出失败原因、错误码、重试建议。右侧展示错误详情与解决方案，避免重复列表字段。可加入堆叠卡片式日志摘要。
```

### Logs 变体
```text
同一设计系统下，渲染 logs 页面。上栏有分类 tab + 过滤器（级别、时间、平台、trace、关键词），下方是日志表格；右侧是所选日志详情（摘要 + json + stack 可折叠）。不重复显示已完成或队列的任务信息。
```

### Settings 变体
```text
同一设计系统下，渲染 settings 页面。左侧为设置分类导航，右侧当前分类详情区。使用下拉和开关优先，不要手填大量自由文本。每组设置有标题+说明+字段。
```

### Toolbox 变体
```text
同一设计系统下，渲染 toolbox 页面。左侧工具卡片网格（图标+标题+摘要），右侧最近工具 + 当前工具说明与打开入口。避免把下载任务详情直接重复放进工具页。
```

## 五、对 image2 的“约束清单”（防重复/防偏题）

1. 不要把日志页变成列表复用页。  
2. 不要把设置页变成下载列表页。  
3. 不要把队列页变成播放器页。  
4. 不要把完成页和失败页共用同一“动作按钮组”。  
5. 同一字段（如文件路径、状态、耗时）在单页内只保留一处权威展示。  
6. 每页必须有“本页独有操作”，且保留统一控件风格。  
7. 图标统一风格、同尺寸，不要混杂系统/emoji风格。  
8. 保持桌面端可读性：高行高、网格对齐、分页控件可点击、滚动自然。

## 六、建议的“最终设计验收标准”（给下一步评审）

- 7 个标签页都能独立复用，切换时骨架/风格不漂移。
- `queue` 作为常用首页能承载启动链路（搜索、起停、计数、目录）。
- 运行态数据、设置态、日志态三类信息分离，不重复渲染同一字段。
- 深浅主题都无明显跳动和对比度问题，移动端也有合理降级。
- 输出图像能直接映射到当前 `app/web/static/*` 的布局分区（左栏、主区、右栏）。

> 这份提示词可用于：设计评审、AI 生成样式探索、或与 UI 施工团队对齐。最终产物建议再用 `app/web/static/app.css` 风格令牌做一次落地校验。
