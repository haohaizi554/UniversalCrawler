# GUI智能换行与底栏布局修复

日期：2026-06-29

## 背景

完成页文件信息区域显示长文件名、保存路径时，部分窗口尺寸下会把路径拆成过多短行，并继续推高页面内容，导致底部分页或状态栏被顶出可视区域。顶部视频数量下拉框也存在控制框按最长选项固定宽度的问题，选中较短项时显得过宽。

## 根因

1. 完成页文件信息直接放在普通布局里，长路径产生的高度会参与整页高度计算，压缩底部分页和全局状态栏空间。
2. `SmartWrapLabel` 原本定义在活动下载页内部，完成页复用不够清晰，后续行为难以统一测试。
3. 全局语言同步会遍历 `QLabel` 并重新调用 `setText(label.text())`。对智能换行标签而言，`label.text()` 是带换行的显示文本，不是原始路径；重新写回后会污染 raw text，造成越刷新越碎的换行。
4. 顶部数量框把控制框宽度和弹层宽度绑定在一起，选中 `50 个视频` 时仍按 `20 个视频（推荐）` 计算按钮宽度。

## 修复

1. 将 `SmartWrapLabel` 抽到 `app.ui.components.smart_wrap_label`，作为 GUI 共享组件。
2. 为智能换行标签标记 `i18nSkipText=true`，语言同步只翻译固定 UI 文案，不再改写文件名、路径、链接等动态文本。
3. 完成页文件信息卡内部改用 `QScrollArea`，长内容只在信息卡内滚动，不再把整页和底部状态栏撑出屏幕。
4. 完成页文件名、保存路径的值标签移除错误的顶部对齐宽度约束，让标签实际占满两列可用宽度后再计算换行。
5. 顶部数量框拆分控制框宽度与弹层宽度：控制框按当前选中项自适应，弹层仍按最长选项保证不截断。

## 回归覆盖

新增和更新的前端契约测试覆盖：

1. 智能换行组件在宽度足够时不会把路径永久拆成过短片段。
2. 完成页文件信息使用内部滚动区，保存路径 raw text 不允许被换行污染。
3. 紧凑高度下各 GUI 页面不会把全局状态栏顶出窗口。
4. 顶部数量框宽度跟随当前选中项变化，弹层宽度仍能容纳最长选项。

验证命令：

```powershell
python -m pytest tests/test_unified_frontend_contract.py -q
python -m pytest -q
```

结果：`66 passed`；全量测试 `1527 passed, 1 skipped`。

## 补充：下拉选中行右侧留白

同日继续排查顶部数量框和分页下拉框时，发现弹层宽度看似足够，但选中行右侧仍出现白色留白。最终复现数据为：`view=176`、`viewport=176`、`popup=188`。真正多出来的是 Qt/Windows 为原生弹层顶层窗口预留的 12px 滚动条槽位；之前只检查和修复 view/viewport 内部，因此无法覆盖 popup 顶层窗口右侧区域。

修复方式：`NoFocusItemDelegate` 绘制选中、悬停背景和主题色边框时，统一使用 popup viewport 的完整行宽；同时将 popup 顶层窗口、view、viewport 全部锁到同一个 `target_width`，full-expand 短列表下把水平/垂直滚动条容器压为 0。顶部数量框恢复为按最长选项计算本体宽度，并让 popup 与本体等宽。

新增像素级回归：打开下拉后抓取 popup 顶层窗口，不只抓 viewport；抽样选中行最右侧像素，确认它是当前主题色，避免“view 内部正常但 popup 右侧露白”的问题回归。
