# GUI弹窗主题与默认打开方式修复

日期：2026-06-29

## 背景

任务清单确认弹窗和默认打开方式弹窗在主题切换后会出现控件色彩不一致：弹窗背景、表格、按钮、复选框可能退回系统默认颜色。默认打开方式入口也存在语义不清的问题，设置页按钮发出的视频/图片选择参数与实际弹窗流程不一致。

## 根因

1. 两个 `QDialog` 主要依赖全局样式和父窗口样式继承，阻塞弹窗创建时可能没有稳定拿到当前主题色。
2. 文件关联弹窗使用原生 `QCheckBox` 和 `QDialogButtonBox`，按钮与勾选图标容易被系统样式或全局成功图标覆盖。
3. 设置页“绑定默认打开方式”按钮发出 `(True, False)`，但主窗口实际忽略该参数并重新弹选择框，代码语义容易误导。

## 修复

1. 新增弹窗局部主题样式，显式设置弹窗、说明文字、表格、滚动条、按钮和选项行颜色。
2. 默认打开方式弹窗改为产品化布局：标题、说明、视频/图片选项、状态提示、取消和绑定按钮。
3. 默认打开方式弹窗改用自绘 `ThemeCheckBox`，避免绿色系统成功图标混入主题色选择框。
4. 设置页绑定入口改为默认请求视频和图片，并继续通过主窗口弹窗让用户确认最终选择。

## 生效说明

样式代码需要重启当前 GUI 进程后加载。默认打开方式本身在 Windows 接受注册和 `UserChoice` 写入后会立即影响之后的系统打开行为；如果 Windows 拦截或仍有待确认扩展，程序会打开 Windows 默认应用设置页，需要用户手动确认。
